Ruby Container Objects:

Comparing Arrays and Hashes

7 January 2015

Much of programming is dealing with collections of objects. For example, you may search through a customer database to pull information on specific order types, or you may sort individual cutomers by location. In Ruby, these objects are typically stored inside two types of containerobjects: arrays, and hashes. Here, I'll breifly describe both the Array and the Hash classes. I'll begin by introducing each class individually before concluding with a brief discussion on how they compare with one another.

Arrays

An array is an ordered collection of objects. Objects within an array are indexed by integers. If n is the length, or the total number of objects in the array, then objects in the array are indexed beginning with 0, and continue incrementally until the final object in the collection, which is indexed by n - 1.
