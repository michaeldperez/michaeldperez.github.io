Ruby Container Objects:

Comparing Arrays and Hashes

7 January 2015

Much of programming is dealing with collections of objects. For example, you may search through a customer database to pull information on specific order types, or you may sort individual cutomers by location. In Ruby, these objects are typically stored inside two types of container objects: arrays, and hashes. Here, I'll breifly describe both the Array and the Hash classes. I'll begin by introducing each class individually before concluding with a brief discussion on how they compare with one another. Since this is only a brief introduction to Ruby's container objects, those interested in a more thorough treatment are referred to "Chapter 9: Collection and Container Objects" in David A. Black's The Well-Grounded Rubyist, and "Chapter 3: Ruby's Building Blocks: Data, Expressions, and Flow Control" in Peter Cooper's Beginning Ruby.

Arrays

An array is an ordered collection of objects. Objects within an array are indexed by integers. If n is the length, or the total number of objects in the array, then objects in the array are indexed beginning with 0, and continue incrementally until the final object in the collection, which is indexed by n - 1. Any object can be stored in an array: strings, numbers, booleans, symbols, even other arrays and hashes.

Arrays can be created one of four ways:

  • By making a call to the new method.
  • Literally, using the square-bracket array constructor.
  • With the Array method.
  • With the special %w{...} notation.
Here I'll only be using the literal notation. Readers interested in learning the other methods for creating arrays are referred to Ruby's online documentation site and/or the references cited above. Using the literal constructor, an empty array can be created as follows: 
          x = []
