Ruby Container Objects:

Comparing Arrays and Hashes

7 January 2015

Much of programming is dealing with collections of objects. For example, you may search through a customer database to pull information on specific order types, or you may sort individual cutomers by location. In Ruby, these objects are typically stored inside two types of container objects: arrays, and hashes. Here, I'll breifly describe both the Array and the Hash classes. I'll begin by introducing each class individually before concluding with a brief discussion on how they compare with one another. Since this is only a brief introduction to Ruby's container objects, those interested in a more thorough treatment are referred to "Chapter 9: Collection and Container Objects" in David A. Black's The Well-Grounded Rubyist, and "Chapter 3: Ruby's Building Blocks: Data, Expressions, and Flow Control" in Peter Cooper's Beginning Ruby.

Arrays

An array is an ordered collection of objects. Objects within an array are indexed by integers. If n is the length, or the total number of objects in the array, then objects in the array are indexed beginning with 0, and continue incrementally until the final object in the collection, which is indexed by n - 1. Any object can be stored in an array: strings, numbers, booleans, symbols, even other arrays and hashes.

Arrays can be created one of four ways:

  • By making a call to the new method.
  • Literally, using the square-bracket array constructor.
  • With the Array method.
  • With the special %w{...} notation.
Here I'll only be using the literal notation because it is the most common among the various array constructors. Readers interested in learning the other methods for creating arrays are referred to Ruby's online documentation site and/or the references cited above. Using the literal constructor, an empty array can be created as follows: 
        
        >> x = []
Literal constructors can also accommodate objects placed within the array at the time of creation. For example: 
        
        >> x = [1,2,'hello','world']
creates an array, assigned to the variable x, which contains four elements: the numbers 1 and 2; as well as two strings: "hello" and "world". Note that asigning the array to the variable x is arbitrary. One could just as easily have assigned the array to the variable a, or the variable spaghetti; It doesn't matter, although in your own code it is a good idea to label your variables in a way that is expressive.

As mentioned above, array objects are index by integers. Therefore, to access an element in an array, all one needs to do is reference the object by it's index in the array. For example, if we wanted to access the element "hello" in array x above, all we would need to do is call the array and reference the "hello" element's index, which in this case is 2 (remember that indexing begins at 0 and since "hello" is the third element in the array it's index is 2): 

        
          >> x[2]
          >> #=> "Hello"
As we can see referencing the the second index in the array x results in "hello" being printed to the terminal (as signified by the "#=>" character).

Notably, array elements can be assigned to specific indicies--or reassigned if an element already exists--in a similar fashion. If we want to add the element "foo" to the array x as it's fourth element, we can do so by simply typing: 

        
          >> x[4] = 'foo'
          >> x
          >> #=> [1, 2, 'hello', 'world', 'foo']
Likewise, if we wanted to change the fifth element from "foo" to "bar", we would apply the same code again, this time assigning the fourth index to "bar" instead of "foo": 
        
          >> x
          >> #=> [1, 2, 'hello', 'world', 'foo']
          >> x[4] = 'bar'
          >> x
          >> #=> [1, 2, ,'hello', 'world', 'bar']
There are other was to add elements to existing arrays, so readers should consult the Ruby documentation and ensure that they are familiar with these techniques.
