Ruby Classes

An Introduction

21 January 2015

Organizing Behaviors

Everything you work with in Ruby is an object and every object is, in turn, an instance of a particular class. For the most part, a class is simply a collection of methods that determine an instance of the class' attributes and capabilities. The purpose of a class is to package a set of behaviors (methods) together so that all instances of that class can be created and can be expected to operate the same way. For example, let's say we created a Student class and that we endow the class with the methods study and take_test. Then, every time we instantiate a new instance of the Student class (more on this below), that object will be able to study and take tests. Any time you plan on having a set of objects that you would like to share a similar set of characteristics and behaviors, you should organize them into classes.

A Substantive Example

Let's say that you live in the small town of Sunnydale, California, which is currently being overrun by vampires, demons, and other forces of darkness. You, as Watcher, must prepare a set of Slayers to combate the evil forces. You would like for all of the slayers you prepare to share the same set of combat skills, so you organize them into a Ruby class.

Each slayer you train knows how to punch, kick, and stake vampires. However, after you train them they still have no experience. Thus, when initialized, their experience is 0. In Ruby, this setup looks like the following: 

    class Slayer
      @@vampires = 100

      def initialize
        @experience = 0
      end

      def punch
        "Pow!"
      end

      def kick
        "Hiya!"
      end

      def stake
        @@vampires -= 1
        @experience += 1
        puts "You staked a vampire!"
        puts "There are now #{@@vampires} vampires left."
        puts "Your experience is now #{@experience}!"
      end
    end

Ruby class names always start with a capital letter. Here, we've defined a class Slayer, that will determine the behavior of all particular slayers that come henceforth. The initialize method, if present, executes each time a new instance of the Slayer class is created. In this case, each slayer is given an experience level of 0 when created. The variable "experience" is preceded by the "at" (@) symbol, to signify that it is an "instance variable". Instance variables are only visible to the object within which they belong. In other words, only a particular instance of a slayer can see his/her experience level.

The other methods possessed by slayers are fairly straightforward. When a slayer punches a vampire, "Pow!" is returned. "Hiya!" follows after a slayer has kicked a vampire. When a slayer stakes a vampire, however, the stake method makes a note of it, decreases by one the total number vampires in Sunnydale, and informs the slayer that his/her experience has gone up a point.

To illustrate, let's say you train two slayers, Buffy, and Kendra. Instances of user-defined classes are created with the new method: 

    >> Buffy = Slayer.new
    >> #=> #<Slayer:0x149b3c @experience=0>
    >> Kendra = Slayer.new
    >> #=> #<Slayer:0x143e6c @experience=0>
